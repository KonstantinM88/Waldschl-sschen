# Заметки от 03.05.2026

## Конвертация видео меню в WebM на продакшене

На обычном Node.js сервере, VPS, выделенном сервере или Docker конвертация видео должна работать, если проект устанавливается через `npm install` или `npm ci`, затем собирается через `npm run build` и запускается через `npm run start`.

Для конвертации используется пакет `ffmpeg-static`. Он устанавливает ffmpeg-бинарник под ОС сервера, а код загрузки меню ищет его в `node_modules/ffmpeg-static`. Также можно вручную указать путь через переменную окружения `FFMPEG_PATH`.

Текущая схема подходит для продакшена на обычном сервере, но нужно помнить:

- папка `public/uploads` должна храниться на постоянном диске;
- в Docker нужно подключить volume для `public/uploads`;
- после redeploy файлы не должны удаляться;
- если используется несколько серверных инстансов, локальные uploads могут быть разными на каждом инстансе.

Для Vercel, serverless или edge-runtime текущую схему не стоит считать надежной:

- ffmpeg может не попасть в runtime;
- конвертация может упереться во временные лимиты serverless-функции;
- локальная папка `public/uploads` не является постоянным хранилищем;
- файлы могут пропасть после redeploy.

Если сайт будет переноситься на Vercel/serverless, лучше вынести медиа в S3, Cloudflare R2, Supabase Storage или Cloudinary, а конвертацию делать через отдельный worker/job.

Короткий вывод: на обычном Node.js сервере текущая реализация подходит; для serverless нужен отдельный подход к хранению и обработке видео.
